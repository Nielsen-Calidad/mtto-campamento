Procedimiento de referencia · PO-MTTO-005

Las OS se generan mediante solicitud del área correspondiente. El responsable de logística registra y asigna número de OS. La codificación es OS-XX-YYYY donde XX indica el área destino (MC: Taller Mecánico · MT: Taller Metalúrgico · CA: Campamento) e YYYY es la numeración correlativa.

CONFIGURACIÓN

Integración con Google Sheets · sincronización en la nube

📋 Instrucciones — Google Apps Script

1. En tu Google Drive, creá una nueva Hoja de Cálculo (Google Sheets) con el nombre que quieras.
2. En el menú Extensiones → Apps Script, pegá el siguiente código y hacé clic en Implementar → Nueva implementación → Aplicación web.
3. En "Quién tiene acceso" seleccioná Cualquier persona y copiá la URL generada.
4. Pegá esa URL en el campo de arriba.

const SHEET_NAME = "OS"; function doGet(e) { const sheet = SpreadsheetApp.getActiveSpreadsheet() .getSheetByName(SHEET_NAME); const data = sheet.getDataRange().getValues(); const headers = data[0]; const rows = data.slice(1).map(row => { const obj = {}; headers.forEach((h, i) => obj[h] = row[i]); return obj; }); return ContentService .createTextOutput(JSON.stringify(rows)) .setMimeType(ContentService.MimeType.JSON); } function doPost(e) { const sheet = SpreadsheetApp.getActiveSpreadsheet() .getSheetByName(SHEET_NAME); const body = JSON.parse(e.postData.contents); if (body.action === "delete") { const data = sheet.getDataRange().getValues(); for (let i = 1; i < data.length; i++) { if (data[i][0] == body.id) { sheet.deleteRow(i+1); break; } } return ContentService.createTextOutput("ok"); } const headers = sheet.getRange(1,1,1,sheet.getLastColumn()).getValues()[0]; // Si no hay encabezados, crear if (!headers[0]) { const cols = ["id","numero","fecha_pedido","tipo","solicitante","area_solicitante", "responsable","area_destino","plazo","fecha_inicio","fecha_fin","duracion_hs", "proximo_servicio","km_actuales","desc_trabajo","trabajo_realizado", "observaciones","materiales","mano_obra","estado","obs_final"]; sheet.getRange(1,1,1,cols.length).setValues([cols]); } const data = sheet.getDataRange().getValues(); const hdrs = data[0]; const row = hdrs.map(h => body[h] !== undefined ? body[h] : ""); // Buscar si existe para actualizar for (let i = 1; i < data.length; i++) { if (data[i][0] == body.id) { sheet.getRange(i+1, 1, 1, row.length).setValues([row]); return ContentService.createTextOutput("updated"); } } sheet.appendRow(row); return ContentService.createTextOutput("created"); }

Flujo de aprobación — PO-MTTO-005 sección 6.7 y 6.8

Una vez finalizado el trabajo, el Responsable de MTTO aprueba y firma la OS. Luego el ejecutante firma y finalmente el solicitante da conformidad del servicio. Solo entonces la OS se considera cerrada (estado: Aprobada).
